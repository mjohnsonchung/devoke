# CLAUDE.md — Devoke Website
> Working brief for the marketing site. Read this before touching any file.

---

## What This Is

The marketing/landing site for the **Devoke** Chrome extension. Its job is to explain what Devoke does, earn trust, and send people to the Chrome Web Store.

This is not a SaaS dashboard, not a docs site, not a blog. It is a **single-purpose editorial landing page** that happens to have a privacy policy.

---

## Extension Summary (what we're selling)

Devoke rewrites sensationalist headlines and social posts into calm, factual language. It runs on X, Reddit, curated news sites, and Google Search. It never hides content — it translates it. Every rewritten post has a one-tap toggle to see the original.

Core tagline: **"The internet, translated."**

---

## Stack

- **Framework:** Astro (static output — zero client JS by default)
- **Styling:** Plain CSS via `src/styles/global.css` — no Tailwind, no CSS-in-JS
- **Hosting:** Vercel (`vercel.json` not needed — Astro adapter handles it)
- **Fonts:** Google Fonts — Cormorant Garamond (display), DM Sans (UI/body)
- **SEO plugin:** `@astrojs/sitemap`
- **Screenshot loop:** Puppeteer (`scripts/screenshot.js`)

## Design System

Identical palette to the extension popup. Do not deviate.

```css
--color-bg:       #0f1c12   /* page background */
--color-surface:  #162019   /* card/section surface */
--color-green:    #7ab88a   /* primary — CTAs, highlights, ◎ */
--color-text:     #b8d4be   /* body text */
--color-muted:    #8aa090   /* captions, hints */
--color-red:      #c4756a   /* "before" panel accent, error states */
```

**Fonts:**
- `'Cormorant Garamond', serif` — all display headings (h1–h3), pull quotes
- `'DM Sans', sans-serif` — body copy, UI labels, captions

**Aesthetic direction: Editorial Forest**
- Large, airy Cormorant Garamond at 80–120px for the hero
- Grain texture overlay on the hero section (CSS SVG filter)
- `clip-path: polygon()` diagonal cuts between major sections — no flat horizontal rules
- Scroll-reveal: `fadeUp` keyframe + IntersectionObserver, one stagger per section
- Ambient ◎ at ~15% opacity behind the hero text — decorative, not interactive
- No purple. No glassmorphism. No gradients toward white.

## SEO Requirements

Every page must have (set via `Base.astro` props):
- `<title>` — unique per page
- `<meta name="description">` — 140–160 chars
- `<link rel="canonical">`
- Full OG block (`og:title`, `og:description`, `og:image`, `og:type`)
- `twitter:card: summary_large_image`

`index.astro` additionally injects a `SoftwareApplication` JSON-LD block.

`sitemap-index.xml` is auto-generated by `@astrojs/sitemap` at build time.

**Heading hierarchy:** One `<h1>` per page. Never skip levels.

## Puppeteer Screenshot Loop

`scripts/screenshot.js` — for iterating on visual design.

```bash
# Terminal 1
npm run dev

# Terminal 2
node scripts/screenshot.js
```

Outputs timestamped folders: `screenshots/YYYY-MM-DDTHH-MM/`
Each run produces: desktop + mobile full-page PNGs, plus hero and before/after section clips.

Do not commit `screenshots/` to version control.

## Coding Standards

- **No `innerHTML` with external content** — use `textContent` or template literals with static strings only
- **No frameworks inside Astro components** — vanilla JS in `<script>` tags only
- **Component props typed** — use Astro's `interface Props` for all component interfaces
- **Comment non-obvious choices** — especially clip-path values, animation timings, selector rationale
- **Fail gracefully** — if a Google Font fails to load, the design should still work with system serif/sans fallbacks
- **No placeholder copy** — every string on the page is final or clearly marked `<!-- TODO -->`
